# Internationalization (i18n)

The project uses `@inlang/paraglide-sveltekit` for i18n.

## Important

- Use the package-local `bun run i18n:generate` script to regenerate Paraglide artifacts in CI or from a clean checkout
- For desktop, `i18n:generate` also owns the required `svelte-kit sync` step because frontend tests depend on synced framework artifacts
- The paraglide folder (`src/lib/paraglide/`) is generated code and should stay out of git
- Do not rely on `bun run build` side effects to make `bun run check` pass

## Adding New Messages

1. Add keys to `messages/en.json`
2. Add the same keys to every `messages/{languageTag}.json` (or copy English as a temporary placeholder), then run `bun dev` / `bun run build` so Paraglide regenerates.
3. Optional — machine-translate via inlang (from `packages/desktop`, with the inlang CLI configured):

```bash
bun run translate:update
# or re-translate everything:
bun run translate:all
```

See `inlang machine translate --help` for flags (e.g. target language).

## Usage in Components

**Always use i18n for all user-facing text:**

```svelte
<script lang="ts">
  import * as m from "$lib/paraglide/messages.js";
</script>

<button>{m.button_submit()}</button>
<p>{m.welcome_message({ name: userName })}</p>
```

## Naming Conventions

- Use snake_case for all i18n keys (e.g., `thread_copy_thread_id` not `threadCopyThreadId`)
- Group related keys with prefixes (e.g., `settings_`, `thread_`, `common_`)
- **Never hardcode text** in components - use i18n keys instead

## Critical - Paraglide Runtime Imports

The paraglide runtime exports `setLanguageTag`, **NOT** `setLocale`. This is a common mistake that breaks the app.

```typescript
// ✅ CORRECT - use setLanguageTag
import { setLanguageTag } from "$lib/paraglide/runtime.js";

// ❌ WRONG - setLocale does NOT exist
import { setLocale } from "$lib/paraglide/runtime.js";  // WILL BREAK!

// ✅ CORRECT - if you need an alias, use "as"
import { setLanguageTag as setParaglideLocale } from "$lib/paraglide/runtime.js";
```

## Available Runtime Exports

- `setLanguageTag(tag)` - Set the current language
- `languageTag()` - Get the current language
- `sourceLanguageTag` - The source language ("en")
- `availableLanguageTags` - Array of all supported languages
- `onSetLanguageTag(callback)` - Register callback for language changes
